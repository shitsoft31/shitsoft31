# Contributing to ShitSoft

First of all, thank you for your interest in contributing to **ShitSoft**. ❤️

Whether you're fixing a typo, reporting a bug, improving documentation, or building a new feature, every contribution helps make the community better.

## Before You Start

Before creating a new issue or pull request, please:

* Search existing issues to avoid duplicates.
* Read the project's documentation.
* Make sure your contribution aligns with the project's goals.

## Reporting Bugs

When reporting a bug, please include:

* A clear description of the problem.
* Steps to reproduce it.
* Expected behavior.
* Actual behavior.
* Screenshots or logs (if applicable).
* Environment information (OS, version, compiler, runtime, etc.).

## Suggesting Features

Feature requests are always welcome.

Please explain:

* The problem you're trying to solve.
* Your proposed solution.
* Any alternative approaches you've considered.

The more context you provide, the easier it is to discuss the idea.

## Pull Requests

Before opening a Pull Request:

* Keep changes focused on a single topic.
* Follow the existing coding style.
* Update documentation if necessary.
* Make sure your code builds and passes any available tests.
* Write clear commit messages.

Small, focused pull requests are preferred over large ones.

## Commit Messages

We recommend following the Conventional Commits specification.

Examples:

```text
feat: add JWT authentication
fix: resolve memory leak in parser
docs: improve installation guide
refactor: simplify HTTP router
test: add parser unit tests
```

## Code Style

Every project may define its own coding conventions.

Unless stated otherwise:

* Write readable and maintainable code.
* Prefer simplicity over cleverness.
* Avoid unnecessary dependencies.
* Keep functions and modules focused on a single responsibility.

## Be Respectful

We want ShitSoft to be a welcoming community for everyone.

Please:

* Be respectful.
* Be constructive.
* Accept feedback professionally.
* Help others learn and grow.

## Questions

If you're unsure about anything, feel free to open a discussion before starting work.

We're happy to help.

---

Thank you for helping make **ShitSoft** better. 🚀
