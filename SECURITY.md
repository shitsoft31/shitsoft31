# Security Policy

## Reporting a Security Vulnerability

At **ShitSoft**, we take security seriously.

If you discover a security vulnerability in one of our projects, please report it responsibly so we can investigate and fix the issue.

Please **do not** create a public GitHub issue for security vulnerabilities. Public disclosure before a fix is available may put users and projects at risk.

Instead, contact us privately at:

📧 shitsoftofficial@gmail.com

Please include:

* A clear description of the vulnerability.
* Steps to reproduce the issue.
* Affected project and version.
* Potential impact.
* Any suggested fixes or mitigations.

## What Happens Next

After receiving a report:

1. We will acknowledge the report as soon as possible.
2. We will investigate and verify the issue.
3. We will work on a fix if the vulnerability is confirmed.
4. We may publish a security advisory after a fix is available.

## Responsible Disclosure

We appreciate security researchers and contributors who help improve our software.

Please give us reasonable time to investigate and resolve vulnerabilities before publicly sharing details.

We will make every effort to:

* Treat reports confidentially.
* Credit researchers who responsibly disclose vulnerabilities (if they wish).
* Avoid unnecessary disclosure of sensitive information.

## Supported Versions

Security updates depend on each project's maintenance status.

Please check the individual project's documentation for supported versions and security update policies.

Thank you for helping keep ShitSoft projects secure. 🔒
